AccountWalletAddressType

Type of wallet address for an AccountWallet

Internal addresses are used by the wallet implementation to recieve change and are not ordinarily shared. External addresses are for receiving payments and are shared with the sender.

AccountWalletAddressType

Type: ("internal" | "external")

AccountWallet

Wallet for a single account derived from a BIP-0044 path

This class internally creates sub-wallets for and facilitates issuing of external (public receiving) and internal (change) addresses. It is up to the client implementation to determine whether addresses have been previously used (ie by scanning the blockchain) and calling setUsedAddressIndex() accordingly.

DerivationPathType

DerivationPath

Class for handling BIP32 derivation paths - a method of specifying a private or public extended key in a key tree.

A derivation path is a string in the format of format "m/n'/n'/n'" where:

m or M is a literal. Lowercase is a private key; uppercase is public key / is a literal indicatating is an optional level (zero or more) n the child key number in the current level ' optionally nominates the suffixed child key as hardened (subscript 'H' is the notation used in the BIP but apostrophe suffix is the public convention)
